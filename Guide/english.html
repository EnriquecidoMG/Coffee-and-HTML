1. Introduction to HTML


What is HTML?
HTML (HyperText Markup Language) is the standard language used to create web pages. It consists of a series of "tags" or "elements" that define the structure and content of a web page. HTML is not a programming language, but a markup language used to organize and structure content on the web.


Basic Structure of an HTML Page
An HTML page has a basic structure that always includes the following parts:



<!DOCTYPE html>  <!-- Declares the document type (HTML5 here) -->
<html>  <!-- Root tag that wraps all content of the page -->
    <head>  <!-- Contains metadata, links to external files, etc. -->
    <title>My Web Page</title>  <!-- Title displayed in the browser tab -->
    </head>
    <body>  <!-- Contains all visible content of the page -->
    <h1>Welcome to My Web Page</h1>  <!-- Main header -->
    <p>This is a paragraph of text.</p>  <!-- A paragraph -->
    </body>
</html>

Why is HTML Important?
HTML forms the skeleton of all web pages. It is the foundation upon which other technologies like CSS (for styling) and JavaScript (for interactivity) are built.

2. Basic HTML Structure and Elements


Main Tags:

  • <html>: The root tag that wraps all the content of the page.
  • <head>: Contains metadata like the title, external CSS files, and scripts.
  • <title>: Defines the title that appears on the browser tab.
  • <body>: Contains all visible content of the page, such as text, images, links, etc.

Common Tags:

  • <h1> to <h6>: Header tags, where <h1> is the most important header.
  • <p>: Defines a paragraph of text.
  • <a>: Defines a hyperlink (anchor). Example: <a href="https://www.example.com">Visit Site</a>.
  • <ul>: Defines an unordered list. Example: 
    <ul>
    <li>Item 1</li>
    <li>Item 2</li>
</ul>
  • <ol>: Defines an ordered (numbered) list.
  • <li>: Defines an item in a list.
  • <img>: Defines an image. It requires the src attribute (source of the image).

3. Creating Content and Structuring


Text:

4. Links and Navigation


Hyperlinks:
Hyperlinks are fundamental for navigation on the web. The <a> tag is used to create links: 

<a href="https://www.example.com">Visit Example</a>
The target="_blank" attribute opens the link in a new window or tab: 
<a href="https://www.example.com" target="_blank">Open in New Window</a>

Internal Links:
Links can point to other sections of the same page using anchors: 

<a href="#section1">Go to Section 1</a>
Later in the page, use the id attribute: 
<h2 id="section1">Section 1</h2>
        


<!-- Hyperlinks are fundamental for navigation on the web -->

<a href="https://www.example.com">Visit Example</a>

<!-- The target="_blank" attribute opens the link in a new window or tab: -->
<a href="https://www.example.com" target="_blank">Open in New Window</a>

<!-- Internal Links -->
<!-- Links can point to other sections of the same page using anchors: -->

<a href="#section1">Go to Section 1</a>

<!-- Later in the page, use the id attribute: -->
<h2 id="section1">Section 1</h2>

5. Using Images


Insert Images:
To insert an image, use the <img> tag. The src attribute specifies the image source, and alt provides alternative text for accessibility: 

<img src="image.jpg" alt="Description of the image" width="300" height="200">

6. Semantic HTML


What is Semantic HTML?
Semantic HTML uses meaningful tags to improve accessibility and SEO. It helps make the content of a web page clearer and easier to understand.


Semantic Tags:
Some of the semantic HTML tags are:

  • <header>: Defines the header of the page or section.
  • <footer>: Defines the footer of the page.
  • <main>: Defines the main content of the page.
  • <article>: Defines an independent article.
  • <section>: Defines a section of the page.
  • <nav>: Defines a navigation block.
  • <aside>: Defines content that is related, but separate from the main content.

Example of a semantic structure:



<header>
<h1>Welcome to My Site</h1>
<nav>
    <ul>
    <li><a href="#home">Home</a></li>
    <li><a href="#about">About</a></li>
    </ul>
</nav>
</header>

<main>
<article>
    <h2>My First Article</h2>
    <p>Article content here...</p>
</article>
</main>

<footer>
<p>© 2024 My Website</p>
</footer>

1. What is CSS?


CSS (Cascading Style Sheets) is a design language used to describe the presentation and style of an HTML or XML document. While HTML is responsible for the structure and content of a webpage, CSS is used to determine its visual appearance: colors, fonts, margins, the arrangement of elements on the page, etc.


Function of CSS:

  • HTML is used to define the basic structure of a webpage, such as the content of headers, paragraphs, links, images, lists, tables, forms, etc.
  • CSS is used to define how that content will appear in the browser. For example, you can change the background color of the page, the font size and style of paragraphs, or align elements in a certain way (e.g., placing elements in a row or a column).

CSS and the Cascade (Cascading)

The term "Cascading" refers to how CSS styles are applied in a hierarchy or cascade. If there are conflicts between style rules, the browser will apply the style rules based on priority. Inline rules (in the style attribute of an element) take precedence over internal or external rules, and more specific rules (such as a class selector) take precedence over more general ones (such as a tag type selector).


2. Basic Structure of an HTML and CSS Project


First, we create an HTML file to place our content. Then, we link an external CSS file that will handle the styling.

  • HTML: The basic structure of the web page.
  • CSS: Contains the styling and visual design.

    <title>Document</title>

3. Understanding CSS Syntax


The basic syntax of CSS is very simple:


selector {
property: value;
}
  • Selector: The HTML element you want to style.
  • Property: The aspect you want to change (such as color, size, etc.).
  • Value: The setting you assign to the property.

4. Applying Basic Styles


Here are some basic CSS examples for applying styles:

  • color: Changes the color of the text.
  • background-color: Sets the background color of an element.
  • margin: Defines the space outside an element.
  • padding: Defines the space inside an element.

/* Applying Basic Styles */

/* 1. Color: Changes the color of the text */
body {
    color: #333; /* Dark gray color for text */
}

/* 2. Background-color: Sets the background color of an element */
h1 {
    background-color: lightblue; /* Light blue background for <h1> elements */
}

/* 3. Margin: Defines the space outside an element */
p {
    margin: 20px; /* 20px margin around each <p> element */
}

/* 4. Padding: Defines the space inside an element */
div {
    padding: 15px; /* 15px padding inside each <div> element */
    border: 1px solid #ccc; /* Adding a border for visibility */
    background-color: #f4f4f4; /* Light gray background for <div> */
}

5. Flexbox


Flexbox is a powerful layout technique to create more flexible and responsive designs. It allows you to align and distribute elements efficiently within a container.

  • justify-content: Aligns elements horizontally.
  • align-items: Aligns elements vertically.
  • flex-direction: Defines the direction in which elements are placed.

6. CSS Grid


CSS Grid Layout is an advanced tool to create more complex page designs. It allows you to divide the page into rows and columns with ease.

  • grid-template-columns: Defines the columns in the grid.
  • grid-template-rows: Defines the rows in the grid.
  • gap: The space between elements in the grid.

/* Grid Layout Example */

/* 1. grid-template-columns: Defines the columns in the grid */
.container {
    display: grid;
    grid-template-columns: repeat(3, 1fr); /* Creates 3 equal-width columns */
    gap: 10px; /* Adds 10px gap between grid items */
}

/* 2. grid-template-rows: Defines the rows in the grid */
.item {
    grid-template-rows: auto; /* Defines the rows' height */
    background-color: lightgray; /* Background color for items */
    padding: 20px;
    text-align: center;
}

7. Media Queries


Media queries allow you to create responsive designs that adapt to different screen sizes (phones, tablets, desktops, etc.).



@media screen and (max-width: 768px) {
/* Style changes for smaller screens */
body {
    font-size: 14px;
}
}

  • max-width: Defines a style that applies when the screen width is less than the specified value.
  • min-width: Defines a style that applies when the screen width is greater than the specified value.

8. CSS Animations


Animations allow elements to change smoothly over time. You can make elements move, change color, and much more.

  • @keyframes: Defines an animation with a set of steps (frames).
  • transition: Allows a style change to occur smoothly when the user interacts with an element.

CSS Glossary (Terms)


Here are some common terms used in CSS and their functions:


Term Definition Function
Selector A CSS selector is used to target an HTML element to apply styles to it. Selectors are used to define which HTML elements should be styled. Example: p, #id, .class.
Property A CSS property defines the aspect of an element that you want to style. Properties define the characteristics you want to modify. Example: color, font-size, margin.
Value A CSS value is the setting you assign to a property to define how the property should behave. Values specify how properties are applied. Example: red, 16px, auto.
Class A class is an attribute that can be assigned to one or more HTML elements to apply shared styles. Classes are often used to apply the same styles to multiple elements. Example: .button for all button elements.
ID An ID is a unique identifier for an HTML element, used for specific styling or targeting with JavaScript. IDs are used to select a single, unique element. Example: #header for an element with the ID "header".
Margin Defines the space outside the element's border. Margins control the space around elements. Example: margin: 20px;.
Padding Defines the space between an element's content and its border. Paddings control the internal space within elements. Example: padding: 10px;.
Background-color Sets the background color of an element. Used to change the background of elements like div or body. Example: background-color: lightblue;.
Display Defines how an element is displayed on the page. The display property controls whether elements are displayed as a block, inline, or flex container. Example: display: flex;.
Flexbox Flexbox is a layout model that allows you to design complex layouts in a simple way. Flexbox allows elements to adjust and align dynamically within a container. Example: display: flex;.
Grid CSS Grid Layout is a two-dimensional layout system for web design. CSS Grid allows for creating layouts with rows and columns. Example: display: grid;.

Tips for Writing Advanced HTML and CSS Code


Here are some useful tips for writing more advanced, maintainable, and efficient HTML and CSS:


  • 1. Use Semantic HTML: Always use semantic HTML elements like <header>, <footer>, <article>, <section>, and <nav> to provide better structure and meaning to your content. This improves accessibility, SEO, and readability of your code.
  • 2. Use CSS Variables: CSS Variables allow you to store values (like colors, fonts, spacing) in one place and reuse them throughout your stylesheet. This makes your code more maintainable and easier to modify.
  • 3. Avoid Inline Styles: While inline styles (style attribute) can be quick to use, they are difficult to maintain. It's better to separate your CSS into an external file for better reusability and readability.
  • 4. Use Flexbox and Grid Layouts: When creating layouts, consider using flexbox and grid. These modern CSS layout systems offer powerful tools for building responsive and flexible designs without the need for floats or complex positioning.
  • 5. Minimize Specificity Conflicts: Avoid using overly specific selectors (e.g., div#container ul li a) that could cause problems later on. Instead, try to keep selectors simple and use classes for styling. This improves maintainability and reduces the risk of styles being overridden unexpectedly.
  • 6. Use BEM (Block, Element, Modifier) Methodology: The BEM naming convention helps create consistent and scalable class names. For example, .button--large could be a button element with a modifier for the large size. This improves readability and avoids class name conflicts.
  • 7. Optimize for Mobile (Responsive Design): Use media queries to adjust your design for different screen sizes. A mobile-first approach is often recommended, where you build the basic layout for small screens and then add specific styles for larger devices.
  • 8. Use CSS Preprocessors (Sass or Less): CSS preprocessors like Sass or Less offer advanced features such as variables, nesting, and mixins, making your CSS more powerful and organized. Preprocessors help reduce repetitive code and improve scalability.
  • 9. Avoid Using !important: The !important rule can override other styles, but it should be used sparingly. Overusing !important can make it difficult to debug your code and can create confusion about which styles are being applied.
  • 10. Comment Your Code: Always comment your HTML and CSS code, especially in larger projects. This helps others (and your future self) understand your logic and intentions behind certain sections or styles.
  • 11. Keep Your CSS DRY (Don’t Repeat Yourself): Try to avoid repeating styles across multiple selectors. If you find yourself repeating a set of properties, consider creating a reusable class or using CSS variables.
  • 12. Optimize Performance: Keep performance in mind when writing CSS. Use efficient selectors, minimize the use of expensive properties like box-shadow, and reduce the number of HTTP requests by combining CSS files and optimizing images.
